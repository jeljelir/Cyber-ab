Active Reconnaissance

Ethical hacking begins with information gathering and target system knowledge. Reconnaissance uses Footprinting, Scanning, and Enumeration to locate and gather information about a target system.

An ethical hacker gathers as much information as possible about a target system during reconnaissance.

An active reconnaissance cyberattack is a sort of computer assault in which the intruder interacts with the system that is the target of the assault in order to acquire information about the system's weaknesses.

  • Collect data
  • Network range
  • Active machines
  • Find ports and access points
  • System fingerprinting
  • Ports services
  • Network-map

Reconnaissance is active and passive. Reconnaissance is an information-gathering task. Both methods of reconnaissance are called passive assaults since they only gather information, not aggressively exploit the victim. Reconnaissance is generally the first step in exploiting a target system.

 

 

Active Reconnaissance: You'll immediately communicate with the computer to get information. Relevant and factual information. Active reconnaissance without authority is risky. If you're caught, the system admin may punish you and track your activity.

Course Curriculum

Footprinting

Footprinting is used to obtain information about a target computer system or network. Passive and active footprinting exist. Passive footprinting is reviewing a company's website; active information collecting is social engineering. Footprinting is the initial phase when hackers collect as much information as possible to intrude into a target system or select what sort of assaults to use.

 

 

During this phase, a hacker may collect:

  • Domain name and IPs
  • Namespaces
  • Employee numbers
  • E-mails
  • etc.

In the next part, we'll describe how to get basic information from any Internet-connected machine or network.

The Domain Name: You may utilize http://www.whois.com/whois to find out a domain's owner, registrar, date of registration, expiration, name server, owner's contact information, etc.

Finding IP: Use ping at the prompt. This command is on Windows and Linux OS.

IP-ranges: Small sites may use a single IP address, while bigger sites have many IP addresses supporting various domains and subdomains. Using American Registry for Internet Numbers, you may get a company's IP addresses (ARIN).

Site History: Archive.org provides a comprehensive history of any website.

Sniffing

Sniffing monitors and captures all network packets using sniffing tools. It's like "tapping phone cables" to hear conversations. It's computer network wiretapping. If an enterprise's switch ports are open, an employee may sniff the network's traffic. Anyone in the same place may join wirelessly or through Ethernet cable to sniff the network's activity. Sniffing reveals both protected and unprotected communication. With the correct settings and protocols, an attacking party may be able to obtain information for additional attacks or other concerns for the network or system owner.

 

 

Sniffables things:

  • Inboxes
  • FTP-passwords
  • Traffics
  • Telnet authentication
  • Configuring routers
  • DNS
  • etc.

A sniffer sets the system's NIC to promiscuous mode to listen to all segment data. Promiscuous mode permits network interface cards (NICs) to accept all network traffic, even if it's not targeted to them. By default, a NIC rejects all unaddressed communication by comparing the Ethernet packet's destination address to its hardware address (MAC). Non-promiscuous mode makes network monitoring and analysis tools harder to utilize for troubleshooting connection problems or traffic accounting.

A sniffer can monitor all NIC traffic by decoding data packets.

Snorting Types:

  • Active
  • Passive

Passive Sniffing: Passive sniffing locks traffic without changing it. Listening-only passive sniffing. Hub-compatible. All ports on a hub get traffic. All network hosts can observe traffic in a hub-connected network. Attackers may simply grab traffic. Hubs are almost outdated. Modern networks utilize switches. Ineffective is passive smelling.

Active Sniffing: Active sniffing not only locks and monitors traffic, but it may even manipulate it. Switch-based networks are sniffed actively. Injecting ARP into a target network to fill the CAM table. CAM tracks which host is on which port.

Affected protocols: TCP/IP was never intended for security, hence it offers little resistance to attackers.

Sniffables things:

  • HTTP:It sends transparent text without encryption and is a genuine target.
  • SMTP:is used to transmit emails. This efficient protocol lacks sniffer protection.
  • Network News Transfer Protocol:It's utilized for all conversations, although data and passwords are delivered in clear text.
  • POP (Post Office Protocol):receives emails from servers. This protocol's sniffer vulnerability may be exploited.
  • FTP (File Transfer Protocol):sends and receives files but offers no security. All the data is plain text that can be sniffed.
  • IMAP (Internet Message Access Protocol):Like SMTP, IMAP is sensitive to sniffer.
  • Telnet:broadcasts everything (usernames, passwords, keystrokes) in clear text, therefore it's readily sniffed.

Sniffers aren't stupid utilities that show live traffic. Save the capture if you wish to study each packet.

Intercepting lawfully: Lawful Interception (LI) is legally sanctioned access to phone conversations or email messages. LI must be authorized for analysis or evidence. LI is a security method that allows law enforcement to access private communications of persons or organizations. Almost all nations have written and approved legal interception laws; standardization bodies are producing LI technological requirements. Infrastructure protection and cyber security are typical LI goals. Unless otherwise forbidden, private network operators may preserve LI capabilities inside their own networks.

Sniffing Tools

There are several programs for sniffing network traffic, and they all have their particular capabilities to assist hackers evaluate data. Sniffing applications are widespread. Here are some intriguing ones.

 

 

Sniffables things:

  • BetterCAP: is a robust, adaptable, and portable tool that can execute MITM attacks, alter HTTP, HTTPS, and TCP traffic in real-time, and sniff for passwords.
  • Wireshark:is a popular packet sniffer. It has several tools to help analyze traffic.
  • Tcpdump:Command-line packet analyzer. It can intercept TCP/IP and other network traffic. tcpdump.org A Windows version of the popular Linux packet sniffer tcpdump, which displays header information.
  • Dsniff:is a collection of sniffing tools intended to intercept and disclose passwords. Dsniff works on Unix and Linux but not Windows.
  • NetWitnessNextGen:is a hardware-based sniffer that monitors and analyzes network data. FBI and other agencies utilize this technology.
  • etc.

Any of these sniffing programs may be used to examine network traffic and deconstruct data

Lab: Network Reconnaissance with Nmap

You will see how to use tools to scan and retrieve information from a targeting system using Nmap.

nmap ("Network Mapper") is a free and open source network exploration and security auditing tool. Though it was designed to quickly scan large networks, we use it in this lab to scan the target host. 

nmap -T4 -sT 192.168.178.102
Parameter Usage Description
-p nmap -p 21,22,443 1.1.1.1 Scan selected port
--top-ports nmap --top-ports 100 1.1.1.1 Scan top 100 ports
-sp nmap -sp 1.1.1.1 Scan and ping target (ICMP)
-sT/-sU nmap -sT 1.1.1.1 Scan TCP or UDP
-oN nmap -oN file.txt 1.1.1.1 Save the result in a file
-A nmap -A 1.1.1.1 Perform OS and service detection
--script nmap --script vuln 1.1.1.1 Run the script 'vuln' on targets
