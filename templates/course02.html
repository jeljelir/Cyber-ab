Passive Reconnaissance

Ethical hacking begins with information gathering and target system knowledge. Reconnaissance uses Footprinting, Scanning, and Enumeration to locate and gather information about a target system.

An ethical hacker gathers as much information as possible about a target system during reconnaissance.

  • Collect data
  • Network range
  • Active machines
  • Find ports and access points
  • System fingerprinting
  • Ports services
  • Network-map

Reconnaissance is active and passive. Reconnaissance is an information-gathering task. Both methods of reconnaissance are called passive assaults since they only gather information, not aggressively exploit the victim. Reconnaissance is generally the first step in exploiting a target system.

 

 

Passive Reconnaissance: This procedure doesn't include a computer. This method gathers data without interfering with target systems.

Metasploit

Metasploit is a strong exploit tool. To find out all you need to know about Metasploit, visit https://www.metasploit.com. You may choose between a commercial or free version. The two variants are almost identical. "Kali Distribution" contains the Metasploit community release and additional ethical hacking tools, which you will use as an Ethical Hacker. It's possible to run Metasploit on Linux, Windows, or Mac OS X as a stand-alone application.

Metasploit Payloads

To put it another way, hackers use scripts called "payload" to connect with a compromised system. They may deliver data to an user's system by using payloads.

In terms of Metasploit payloads, there are three basic kinds:

 

 

  • Singles: A single is a basic and simple way to begin a conversation and then go to the next level of the relationship. Creating a user, for example, is an example.
  • Staged:In order for an attacker to upload a large file onto a victim's computer, they must first stage the assault.
  • Stages:modules that download the payload components known as stages. Meterpreter and VNC Injection are only two examples of the extensive functionalities available via the different payload stages.

 

 

Lab 1: Whois

Passive recon entails gathering information about the target from sources other than the target itself. Passive recon can be aided by these helpful resources.

Whois employs the RFC 3912 query and answer protocol to search for an object in a database. When a privacy service is used, we can sometimes get "incorrect" information, such as contact information, registrar information, registration date and expiration date, as well as contact email addresses.

Whois of Google will reveal the following information: 

whois google.com

It provides some information about the target domain including:

  • When the domain target is registered
  • What is the registrar of the target domain
  • Which company is target using for name servers
Lab 2: NSLookup / Dig

DNS servers can be queried using nslookup (Name Server Lookup) to determine the IP address associated with a domain name and other DNS data. The ability to pass arguments and use it in a non-interactive manner is available.

The -type parameter can be used to specify the type of query. For Dig we do not use any parameter. 

nslookup -type=A google.com
## or using dig
dig google.com A

It provides some information about the target domain including:

  • All DNS records including TXT, MX, A, AAA, etc.
Lab 3: DNSDumpster

In addition to providing important information, the tools above aren't geared for subdomain enumeration. Useful information and susceptible servers can be found in subdomains. Using DNSDumpster.com, you may look up subdomains and their associated IP addresses, and then try to pinpoint their exact location.

Lab 4: Shodan

Shodan.io is a tool we utilize to dig further into the networks we're interested in. Because we aren't connecting to the devices directly, Shodan acts as a search engine for their online presence and provides a wealth of data throughout our passive investigation.
