Exploitation

Exploitation is a tool or script that allows hackers to exploit a system's weaknesses. Hackers uncover vulnerabilities using Nessus, Nexpose, OpenVAS, etc.

Two kinds of exploits exist:

  • Remote Exploits:These exploits involve a remote system or network. Distant exploits let hackers access remote systems.
  • Local Exploits:are utilized by a local system user who wishes to exceed his permissions.

An attacker or pentester might exploit a flaw or vulnerability to produce undesirable or unexpected behavior, such as gaining control of the target system, escalating privileges, launching a denial-of-service attack, or stealing sensitive data.

Course Curriculum

Exploitation Types

Payload: a pice of code that remotely execute.

Shellcode: Small bit of script used as the payload; often opens a command shell where the adversary may manage the compromised computer. Since penetration testing can identify simple shellcodes being transmitted over the network, they are commonly encoded to prevent detection.

Bind shell: Bind shells aggressively wait for connections on a certain port when executed. This port gives the attacker shell access.

Reverse shells: A reverse shell, sometimes called a remote shell or "connect-back shell," exploits the target system's vulnerabilities to access the victim's computer. Connect to a distant computer and redirect the target system's input and output connections so the attacker may access it remotely.

Pivoting: Metasploit uses pivoting to shift communication from a compromised machine to other, inaccessible networks.

 

 

DDoS

A DDoS attack overloads an internet service or website with traffic from various sources to render it inaccessible. A DDoS assault employs multiple computers and Internet connections, frequently spread internationally in a botnet, to overwhelm a targeted site with packets. A volumetric DDoS assault may create tens or hundreds of Gigabits per second. Your typical network can't manage such load, we're sure.

 

Types of DDos

Volume-based Attacks: Volume-based assaults include TCP/UDP/ICMP/spoofedpacket floods. Layer 3&4 Attacks. An attacker attempts to overwhelm the target's bandwidth. Bits per Second measure assault magnitude.

Protocol Attacks: A protocol attack may involve SYN flooding, Ping of Death attacks, fragmented traffic attacks, Smurf DDoS, and more. Server resources, as well as those of firewalls and load balancers, are depleted in this kind of assault. Packets per Second (PPS) is the unit of attack magnitude.

Application Layer Attacks: gAttacks on the Application Layer include Slowloris, Zero-day DDoS, DDoS attacks targeting Apache and Windows or OpenBSD vulnerabilities, and others. Web servers are the target of this attack. Requests per Second (RPS) are used to quantify the severity of an assault.

 

 

SQL Injection

It is possible to include SQL commands into URL strings or data structures to get a desired answer from the data that are linked to online applications. Typically, this form of attack occurs on websites built with PHP or ASP.NET.

 

How could be done

  • To completely remove a system's database, one must do a full database dump.
  • To make changes to the databases' content.
  • To check that are not permitted by the application.

When an application doesn't adequately check the inputs before delivering them to a SQL query, this sort of attack is possible. There are a number of places where injections may be found. You may check whether a SQL injection attack is possible by inserting the "'" character into a string and seeing if an error occurs.

 

 

Lab: LFI vulnerability Exploitation

Enter the following command in your Linux terminal to get the tool

git clone https://github.com/mzfr/liffy.git

Then we run the tool with the following command.

cd liffy.git

Now run Liffy using Python

python liffy.py

You will see the guide switches in the output, you can use these switches to scan your desired site and find the LFI vulnerability, for example, pay attention to the following command

python liffy.py –url http://site.com/lfi-google.php?file= –filter

Using the -url switch, we have introduced the vulnerable site's input, and with the -filter switch, we will be able to read the file. There are various switches, some of which we will introduce:

  • expect, input, data: to execute the command
  • filter: reading files
  • cookies: cookie and session change
  • location: the address of the files
