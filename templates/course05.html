Vulnerability Assessment

A computer system vulnerability is a fault or weakness that might be exploited to inflict harm or enable an attacker to influence the system. This is distinct from a "cyber threat" since network assets (such as a computer, database, or application) already have computer system vulnerabilities. Cybercriminals employ these weaknesses in their assaults, therefore some people use the phrases interchangeably. Computer vulnerabilities are exploited based on their nature and the attacker's motivations. These vulnerabilities might result from unforeseen software interactions, system components, or program faults. Modern network design is so sophisticated that it's impossible to find and fix all vulnerabilities. Knowing common network vulnerabilities and how to resolve them helps lessen the chance of a data breach. Here are some kinds of security vulnerabilities:

Computer security vulnerabilities may be categorized by location, cause, and usage. These vulnerabilities include:

Network vulnerabilities: Hardware or software flaws expose a network to outside infiltration. Insecure Wi-Fi and firewalls are examples.

OS vulnerabilities: These are OS vulnerabilities that hackers may exploit to gain access to or destroy an asset. Superuser accounts and hidden backdoor programs are examples.

Humans Vulnerabilities: are often the weakest link in cybersecurity. User mistakes may disclose valuable data, provide attack sites, or interrupt systems.

Process vulnerabilities: Some process controls generate vulnerabilities (or a lack thereof). Example: weak passwords (which may also fall under human vulnerabilities).

Software flaws: Log4j has zero-day vulnerabilities that can be exploited. These vulnerabilities are sometimes significant and need prompt response.

A computer system vulnerability is a fault or weakness that might be exploited to inflict harm or enable manipulation. These vulnerabilities might result from unforeseen software interactions, system components, or program faults.

 

 

Course Curriculum

Lab 1: Vulnerability Assessment with OpenVAS

We are not going to cover the installation of OpenVAS. You can easily google it.

OpenVAS is a free and open-source framework of services and tools that provides a comprehensive vulnerability scanning and vulnerability management solution. OpenVAS has been installed and configured for you in our Kali Linux image.

Login to OpenVAS with the password you set during the installation.

Click on the wizard and set the IP address or the hostname of the target and start the scan.

You can also create an advanced scan to have more leverage on parameters.

You will see the progress of the scan.

If you click on the scan, you will see found vulnerabilities scored based on CVSS of CVEs.

And if you click on any item on the list, you will see details.

Lab 2: Vulnerability Assessment with Nikto

We are not going to cover the installation of Nikto. You can easily google it.

Nikto is a simple, open-source web server scanner that examines a website and reports any vulnerabilities that could be exploited or hacked. It's also one of the most widely used website vulnerability tools in the industry, and is widely regarded as the industry standard in many circles.

By using the parameter -h, we can define the IP address or the hostname of the target. 

nikto -h google.com

You can also analyze the SSL certificate of the host by using the parameter -ssl. 

nikto -h google.com -ssl

Here is the scan of scanme.nmap.org that doesn't support SSL. 

nikto -h scanme.nmap.org
